================================================================================
HOW DARK MODE WORKS
================================================================================

The CSS uses CSS custom properties that automatically update in dark mode.

To toggle dark mode, add/remove the "dark" class from the <html> element:

JavaScript:
  // Toggle dark mode
  document.documentElement.classList.toggle('dark');
  
  // Enable dark mode
  document.documentElement.classList.add('dark');
  
  // Disable dark mode
  document.documentElement.classList.remove('dark');

HTML:
  <html class="dark">  <!-- Dark mode enabled -->
  <html>               <!-- Light mode -->

The CSS variables defined in :root and html.dark automatically adjust all colors.

  Border opacity: border-accent/40
  Ring opacity: ring-accent/20
  Gradient opacity: from-black/60, from-warning/10

PSEUDO-CLASSES:
  :focus for focus states (focus:outline-none:focus, focus:ring-1:focus, etc.)
  :hover for hover states (hover:bg-accent/10:hover, etc.)
  ::placeholder for placeholder styles (placeholder-txt-tertiary::placeholder)

ESCAPE SEQUENCES:
  Slashes in class names are escaped: w-1/2 becomes .w-1\/2
  Colons in variants are escaped: focus:ring becomes .focus\:ring:focus

================================================================================
TROUBLESHOOTING
================================================================================

Q: Styles aren't applying?
A: 1. Check that the CSS file is linked in <head>: <link rel="stylesheet" href="tailwind-static.css">
   2. Verify the file path is correct (relative to HTML file)
   3. Check browser console for 404 errors
   4. Clear browser cache and reload
   5. Check that classes are spelled correctly

Q: Dark mode not working?
A: 1. Verify you're adding "dark" class to <html> element
   2. Check DevTools that class is actually applied
   3. Test with: document.documentElement.classList.toggle('dark')
   4. Verify CSS variables are defined (inspect html.dark in DevTools)

Q: Colors look wrong?
A: 1. Check DevTools computed styles for the element
   2. Verify --color-* CSS variables are showing correct values
   3. Check that color classes match your custom palette
   4. In dark mode, verify html.dark rules are applied

Q: Font not working?
A: 1. Ensure Google Fonts link is in <head> before CSS link
   2. Check that font names in CSS match loaded fonts
   3. Verify font-family is actually applied via DevTools

Q: Responsive classes not working?
A: The CSS doesn't include responsive prefixes (md:, lg:, etc.)
   If you need responsive styles, add new utilities manually.

Q: Missing a utility class?
A: 1. Check TAILWIND-CSS-README.md for the list of utilities
   2. If not listed, manually add it to tailwind-static.css
   3. Include both light and dark variants if needed
   4. Update TAILWIND-CSS-README.md with the addition

================================================================================
ADDING NEW UTILITIES
================================================================================

If you add new Tailwind classes to your HTML, add them to tailwind-static.css:

EXAMPLE: Adding rounded-xl

  .rounded-xl { border-radius: 0.75rem; }

EXAMPLE: Adding text-base

  .text-base { font-size: 1rem; line-height: 1.5rem; }

EXAMPLE: Adding w-32

  .w-32 { width: 8rem; }

EXAMPLE: Adding focus:ring-2

  .focus\:ring-2:focus { box-shadow: var(--tw-ring-inset) 0 0 0 calc(2px + var(--tw-ring-offset-width)) var(--tw-ring-color); }

EXAMPLE: Adding dark mode variant

  .dark .bg-custom { background-color: #custom-dark-value; }

For opacity modifiers:
  bg-accent/50 → .bg-accent\/50 { background-color: rgb(59 130 246 / 0.5); }
