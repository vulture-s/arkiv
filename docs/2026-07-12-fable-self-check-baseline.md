# Fable self-check baseline audit — 2026-07-12

A full-surface adversarial self-check of the arkiv backend (`server.py`, `ingest.py`, `db.py`, `export.py` and siblings). Twenty-six findings survived adversarial verification — each was reproduced at the code level (many with a live repro), not inferred. Zero findings remained merely plausible. The dominant theme is **one 4290-line god-file with ~80 routes and per-process module-global state**: the audit-H3 single-flight ingest guard is bypassed by three separate heavy pipelines (bin-copy, retranscribe-all, server-side ingest timeout-orphans), the same SQL/format logic is copy-pasted 3–4×, and the most destructive routes carry zero test coverage.

Rankings below use the stated cost function — security findings by exploitability × blast radius, quality findings by change-cost × bug-likelihood. Each row maps to a hardening PR: **PR1**=security round-4 (path leaks + `db.py:124` guard), **PR2**=ruff, **PR3**=coverage, **PR4**=mypy, **PR5**=dep-audit, **PR6**=frontend, **PR7**=state/constants extraction, **PR8–11**=router split.

## Round priorities

Round 4 should lead with **PR1 (security round-4)** — it closes the two write-primitive gaps that are genuine exploitability × blast-radius outliers (**#1** offload arbitrary-write / RCE and **#4** the CSRF-open ChromaDB `rmtree`), plus the cheap high-value fixes that share its surface (**#3** Windows export-allowlist split, **#22** projects path-leak, **#10** retranscribe language validator). Run **PR3 (coverage)** immediately alongside it, because five of the highest findings are security controls that exist but have **no regression net** (**#4, #5, #19, #20, #21**) — shipping PR1's guards without pinning them just resets the same clock. Then take **PR2**, consolidating the four H3-guard bypasses and the three orphaned-child / partial-file resource leaks (**#2/#6/#7/#11/#12/#13/#15**) into one shared `run_ingest_subprocess` helper — this is the natural seam that de-risks the **PR8–11 router split**, so land it before the split rather than forking the guards per router. **PR7** (state/constants extraction: split-brain `DB_PATH` **#9**, the duplicated SQL/timecode/subtitle/ext logic **#14/#16/#17/#18**, and the `runtime_state` module **#26**) is the enabling refactor that makes the monolith split safe. Defer **PR4/PR5/PR6** — no finding this round is blocked on mypy, dep-audit, or frontend, and the mono-file (**#8**) is better addressed structurally by PR7→PR8–11 than by lint passes.
